Store in a vector database (e.g., ChromaDB, FAISS, Pinecone, or Qdrant)
Step 3 — Build the Retrieval Engine (RAG)
When a query arrives:

Embed the query/JD text
Do semantic similarity search against the vector store
Retrieve top 5–10 most relevant assessments
Step 4 — Build the API
